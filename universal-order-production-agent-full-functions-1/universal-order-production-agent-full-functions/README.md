# Universal AI Order & Production Assistant

Локальна MVP/demo ERP-система для ФОП: прийом замовлень, асортимент, техкарти, склад, закупки, виробництво, залишки, звіти, Telegram-бот і AI-асистент.

Поточне демо налаштоване під кондитерську, але структура зроблена універсальною: завтра це може бути майстерня, виробництво декору, кухня, цех або будь-який бізнес, де є матеріали, замовлення і виконання.

## Що вже працює

- Web-кабінет для реєстрації, входу, налаштувань і роботи менеджера.
- Demo cakes workspace з товарами, техкартами, матеріалами, залишками, закупками, demo-фото товарів і тестовими сценаріями.
- Повний складський цикл: матеріали, мінімальні залишки, партії FIFO, резерви, доступна кількість, списання після виробництва.
- Контроль залишків і план закупівель: система показує, чого немає, що нижче мінімуму, що потрібно під найближчі замовлення.
- Створення чернетки заявки на закупівлю з плану дефіциту.
- Прийом закупки з формуванням партій на складі.
- Замовлення: створення, розрахунок ціни, собівартості, знижок, кастомізацій і резервування матеріалів.
- Виробництво: взяти в роботу, завершити зі списанням резерву, повернути резерв.
- Звіти складу: остаточний баланс на дату і помісячні різниці руху матеріалів.
- Табличні частини з Excel-подібною навігацією, валідацією і українськими підказками.
- Telegram-бот для клієнтських замовлень і менеджерських команд.
- Бот-флоу з фото-прикладами, алергенами, доставкою, оплатою, Новою Поштою як planned-збором даних.
- Два AI-режими: економний режим і повний асистент з обмеженням прав.
- AI-асистент може працювати тільки з бізнес-діями користувача. Він не має права змінювати код, файли, схему, `.env`, залежності або деплой.

## Що planned / ще не production

- `Платежі` у web уже є як неактивний planned-блок. Логіка оплат і знижок частково є на backend/бот-рівні, але повний фінансовий web-екран ще не активований.
- `Нова Пошта` поки збирає місто, відділення/поштомат, ПІБ і телефон отримувача, але не створює ТТН і не викликає API.
- Full AI Assistant потребує окремого живого тестування в браузері й боті перед реальним запуском.
- Це локальна demo/MVP-система, не публічний production-сайт.

## Головний demo-цикл

1. Запустити web-кабінет.
2. Зареєструватися або увійти.
3. Натиснути `Demo cakes`.
4. Перевірити асортимент і техкарти в `Операції`.
5. Відкрити `Склад` і переглянути залишки, резерви, мінімальні пороги та план закупівель.
6. Натиснути `Створити заявку` у плані закупівель.
7. Перейти в `Операції` -> `Закупки`, перевірити чернетку і прийняти закупку.
8. Створити тестове замовлення або оформити замовлення через Telegram.
9. Відкрити замовлення, взяти його в роботу.
10. Завершити виробництво, щоб система списала зарезервовані матеріали.
11. Перевірити, як змінилися залишки, резерви, доступна кількість і сальдо у звітах.

У demo-даних спеціально є матеріали, яких достатньо, матеріали нижче мінімуму і матеріали з нульовим залишком. Це зроблено, щоб контроль закупівель було видно одразу під час презентації.

## Швидкий запуск для користувача

Основна інструкція для тестувальника без програмування:

```text
BOT_TEST_INSTRUCTION_UA.txt
```

Коротко:

1. Запустити web-кабінет:

```text
START_APP.bat
```

2. Відкрити:

```text
http://localhost:3000
```

3. Зареєструвати користувача.

4. Для Telegram demo заповнити:

```text
TELEGRAM_BOT_TOKEN
MANAGER_CHAT_ID
OPENROUTER_API_KEY
```

5. Натиснути `Demo cakes`.

6. Запустити Telegram-бота з web-кабінету: `Операції` -> `Керування ботом` -> `Запустити бота`.

Для постійної роботи бота на локальному Windows-комп'ютері після перевірки ручного запуску натисніть у web `Увімкнути постійно`. Запасний спосіб через папку:

```text
install-bot-autostart.bat
```

Він створює задачу Windows `AIOperationsTelegramBot`, яка стартує watchdog при вході користувача в систему. Watchdog піднімає Telegram-бота і перезапускає його, якщо процес зупинився. У web це вимикається кнопкою `Вимкнути постійно`. Запасний спосіб:

```text
uninstall-bot-autostart.bat
```

7. У Telegram написати боту:

```text
/start
```

Приклад клієнтського замовлення:

```text
Хочу шоколадний торт 2 кг на завтра, більше шоколаду, без горіхів, доставка Нова Пошта
```

## Менеджерські команди Telegram

Команди треба писати з Telegram-акаунта, chat id якого вказаний у `MANAGER_CHAT_ID`.

```text
/orders
/order ORD-XXXX
/products
/components
/recipe Chocolate Cake
/stock
/purchases
/receive_purchase PR-XXXX
/ready ORD-XXXX
/pickup ORD-XXXX
```

Знижки:

```text
/discount_over_amount 1500 10
/discount_every_n 10 15
/discounts
/discount_disable DISC-XXXX
```

Ручна ціна:

```text
/set_order_price ORD-XXXX 1200 індивідуальна домовленість
```

## AI-режими

Економний режим:
- AI допомагає з розбором повідомлень, підказками, читанням інформації та аналізом.

Повний асистент:
- після підключення підписки може виконувати дозволені бізнес-дії через текст або голос;
- може показати замовлення, прочитати рецепт, змінити статус, створити закупку, проаналізувати склад;
- не може змінювати код, файли, схеми, залежності, `.env` або деплой.

## Архітектура

Бізнес-логіка винесена в JS services. Telegram, web або майбутній n8n-шар мають викликати сервісні функції, а не працювати напряму з таблицями.

Базовий потік:

```text
Telegram / Web / AI -> JS business services -> Data Layer -> Local JSON / Google Sheets
```

Для локального MVP використовується `LocalJsonStore`. Пізніше його можна замінити на `GoogleSheetsStore` без переписування бізнес-логіки.

## Безпека даних

Файл `.env`, локальні логи, release-архіви, web auth storage і secret json файли ігноруються через `.gitignore`.

Не комітьте реальні токени, API keys або приватні credentials у репозиторій.

## Статус

Це робоча локальна demo/MVP-база для презентації та наскрізного тестування циклу ФОП: матеріали -> закупка -> замовлення -> резерв -> виробництво -> списання -> видача -> залишки і звіти.

Перед реальним production-запуском потрібно окремо пройти живе end-to-end тестування, активувати фінансовий блок і підключити реальні інтеграції.
