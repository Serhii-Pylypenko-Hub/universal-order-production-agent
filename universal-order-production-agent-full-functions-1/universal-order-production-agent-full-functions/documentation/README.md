# Документація проєкту

Ця папка містить робочі інструкції до локальної demo/MVP ERP-системи для ФОП.

Система вже дозволяє показати наскрізний цикл:

```text
матеріали -> закупка -> замовлення -> резерв -> виробництво -> списання -> видача -> залишки -> звіти
```

## Поточний стан

Готово для demo-презентації:

- web-кабінет менеджера;
- demo-асортимент з локальними зображеннями товарів;
- техкарти і матеріали;
- склад, FIFO-партії, резерви і мінімальні залишки;
- план закупівель і створення чернетки заявки;
- прийом закупки з оновленням залишків;
- створення замовлення і резервування матеріалів;
- виробництво, завершення і списання;
- звіти складу;
- Telegram-бот для клієнта і менеджера;
- економний AI-режим і підготовлений Full Assistant режим.
- web-кнопки для запуску, зупинки, увімкнення і вимкнення постійного режиму Telegram-бота.

Planned / ще не production:

- web-блок `Платежі` поки неактивний;
- Нова Пошта поки тільки збирає дані клієнта, без створення ТТН;
- Full Assistant треба окремо тестувати в живому сценарії перед реальним запуском.

## Правило для наступних змін

Коли змінюється бізнес-логіка, треба оновлювати разом:

- код;
- схему;
- demo-дані;
- тести;
- `docs/user_guide.md`;
- README або коротку інструкцію, якщо зміна впливає на запуск чи сценарій користувача.

Бізнес-логіка має залишатися в `app/js/*` services. Web, Telegram і AI повинні викликати ці сервіси через API/сервісний шар, а не дублювати правила напряму.
