# Telegram Cakes Demo

Цей сценарій показує, як протестувати Telegram-бота разом із web-кабінетом, складом і виробництвом.

## Передумови

1. Запустити web:

```text
START_APP.bat
```

2. Відкрити:

```text
http://localhost:3000
```

3. Зареєструватися або увійти.
4. Заповнити `TELEGRAM_BOT_TOKEN`, `MANAGER_CHAT_ID`, `OPENROUTER_API_KEY`.
5. Натиснути `Demo cakes`.
6. Запустити бота з web-кабінету: `Операції` -> `Керування ботом` -> `Запустити бота`.

`START_TELEGRAM_BOT.bat` лишається запасним способом запуску, якщо web-кнопка недоступна.

Для постійної локальної роботи після перевірки ручного запуску натисніть у web `Увімкнути постійно`. Це створює Windows-задачу `AIOperationsTelegramBot` і запускає watchdog, який піднімає Telegram-бота після зупинки. Щоб повністю вимкнути постійний режим, натисніть `Вимкнути постійно`.

## Клієнтський сценарій

Клієнт пише боту звичайним текстом українською. Бот збирає:

- продукт;
- вагу або кількість;
- дату;
- побажання до рецепта;
- алергени або обмеження;
- спосіб отримання;
- дані Нової Пошти, якщо клієнт вибрав цей спосіб;
- спосіб оплати;
- контакт.

Приклади:

```text
Хочу шоколадний торт 2 кг на завтра, більше шоколаду і без горіхів
```

```text
Потрібен медовик 1.5 кг на п'ятницю, самовивіз, передоплата
```

```text
Berry Cheesecake 1 кг на 20 число, більше ягід
```

```text
Хочу Napoleon Cake 2 кг на суботу з написом Happy Birthday
```

```text
Хочу шоколадний торт 2 кг, доставка Нова Пошта, Київ, відділення 12, Олена, +380501234567
```

```text
Хочу, щоб менеджер передзвонив мені
```

Нова Пошта в demo тільки збирає дані. ТТН автоматично не створюється.

## Фото і приклади

У demo є локальні зображення товарів:

- Chocolate Cake;
- Honey Cake;
- Berry Cheesecake;
- Napoleon Cake;
- Carrot Cake;
- Cupcake Box.

Bot media seed підв'язаний до цих зображень, тому сценарій з фото-прикладами не порожній. У реальному бізнесі ці файли можна замінити на справжні фото товарів.

## Побажання до рецепта

Для Chocolate Cake demo підтримує:

- `Extra chocolate` - додає Cocoa у розрахунок;
- `Add nuts` - додає Nuts;
- `Remove nuts` - прибирає Nuts із рецепта;
- `Add raspberry` - додає Raspberry;
- `Add inscription` - додає напис.

Ці зміни проходять через `CustomizationService`, тому впливають на BOM, собівартість, ціну, години роботи і залишки.

## Що має отримати менеджер

Після створення замовлення менеджеру приходить Telegram-повідомлення:

- номер замовлення;
- клієнт і контакт;
- дата;
- статус;
- сума;
- доставка;
- оплата;
- позиції замовлення;
- команда для перегляду.

Якщо клієнт просить дзвінок або допомогу менеджера, створюється handoff і менеджер отримує alert.

## Менеджерські команди

Команди треба писати з Telegram-акаунта, chat id якого вказаний у `MANAGER_CHAT_ID`.

```text
/orders
/order ORD-XXXX
/products
/components
/recipe Chocolate Cake
/stock
/purchases
/receive_purchase PR-XXXX
/ready ORD-XXXX
/pickup ORD-XXXX
```

Знижки:

```text
/discount_over_amount 1500 10
/discount_every_n 10 15
/discounts
/discount_disable DISC-XXXX
```

Ручна ціна:

```text
/set_order_price ORD-XXXX 1200 індивідуальна домовленість
```

## Повний тестовий маршрут

1. У web відкрити `Склад`.
2. Перевірити дефіцитні матеріали в плані закупівель.
3. Натиснути `Створити заявку`.
4. Відкрити `Операції` -> `Закупки`.
5. Прийняти закупку.
6. У Telegram створити замовлення.
7. У web або Telegram відкрити замовлення.
8. Взяти замовлення у виробництво.
9. Завершити виробництво командою `/ready ORD-XXXX` або кнопкою у web.
10. Перевірити в `Склад`, що резерв і залишки змінилися.
11. Перевірити звіти: остаточний баланс на дату і помісячні різниці.

## Обмеження demo

- Платежі у web поки неактивні.
- Нова Пошта не створює ТТН.
- Full AI Assistant ще треба тестувати окремо в живому сценарії.
- Це локальна demo/MVP-система, не production-сайт.
